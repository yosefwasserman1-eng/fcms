# 🏛️ The Corinthian Archive (הארכיון הקורינתי)

## 👁️ החזון (The Vision)
הארכיון הקורינתי אינו רק מאגר מידע; הוא **מערכת עצבית מרכזית** לניהול ידע הנדסי, תיעוד טכנולוגי וקהילת מומחים. החזון הוא ליצור ישות דיגיטלית אחת המאחדת נרטיבים טקסטואליים (WordPress), תיעוד טכני מעמיק (MediaWiki), וזירת דיונים תוססת (NodeBB) – הכל תחת קורת גג אחת, מאובטחת על ידי מערכת ניהול זהויות מרכזית (IAM) ונשלטת על ידי ליבה (Core) חכמה.

**מעבר לשימוש אישי: פלטפורמה אוניברסלית**
הפרויקט לא נועד להיות פתרון סגור לאדם אחד. השאיפה הגדולה היא להפוך את הארכיון ל-**Unified Community Stack** (מערך קהילתי מאוחד) – תשתית "מוכנה מהקופסה" שכל ארגון או קהילת יוצרים יוכלו לפרוס עבור עצמם. המטרה היא לספק ארכיטקטורה גמישה ואוניברסלית, ולמנוע מקהילות את הצורך להמציא מחדש את הגלגל בחיבור מערכות מסורבלות.

## 🏗️ ארכיטקטורת המערכת (System Architecture)
המערכת מבוססת על תפיסה של **Decoupled Architecture** (ארכיטקטורה מופרדת), שנועדה לאפשר את פריסת ה-Unified Community Stack בצורה חלקה ונקייה.

### המרכיבים העיקריים:
1.  **The Core (Node.js/Express):** ה"מוח" והנתב הראשי. מנהל את הלוגיקה העסקית, ומתקשר עם כל תת-המערכות באמצעות אדפטרים ייעודיים.
2.  **Identity & Access Management (IAM):** שומר הסף של הארכיון (כדוגמת Keycloak). מספק **Single Sign-On (SSO)** לכל רכיבי הסטאק.
3.  **WordPress (Headless CMS):** מנוע ניהול התוכן לפריטי ארכיון, מופעל "ללא ראש" ומשמש כמחסן נתונים עוצמתי.
4.  **MediaWiki Layer:** מערכת לתיעוד ידע קולקטיבי, עצי מוצר ונהלים.
5.  **NodeBB (Community & Discussions):** מנוע הפורומים והדיונים הקהילתיים.
6.  **Dockerized Infrastructure:** הפרויקט כולו ארוז בקונטיינרים, המאפשרים פריסה מיידית בכל שרת.

## 🧩 תוספים מלווים והתאמת פלטפורמות (Companion Plugins)
כדי שהפלטפורמות הללו (וורדפרס, מדיה-ויקי וכו') לא יתפקדו כמערכות מבודדות אלא כחלק מסטאק אחיד, הפרויקט כולל פיתוח של **תוספי אינטגרציה ייעודיים** ("Companion Plugins") לכל מערכת.
במקום לפרוץ או לשנות את קוד המקור של המערכות (מה שיקשה על עדכונים), התוספים המלווים מותקנים עליהן כהרחבות רשמיות ותפקידם:
* **The Corinthian WP Bridge:** תוסף לוורדפרס שחושף שדות Metadata ייעודיים ל-API, מכבה יכולות UI מיותרות (כופה מצב Headless נקי), ומחבר את וורדפרס לאימות של מערכת ה-IAM המרכזית.
* **The Corinthian Wiki Extender:** הרחבה למדיה-ויקי שפותחת Webhooks ומאפשרת ל-Core ליצור דפי ויקי באופן אוטומטי ובפורמט קבוע, תוך סנכרון הרשאות משתמשים.
* הגישה הזו מבטיחה שהמערכות מתיישרות לפי הצרכים של הארכיון הקורינתי, ולא להפך.

## 🔌 פילוסופיית האדפטרים (The Adapter Pattern)
בלב הסטאק האוניברסלי עומד עקרון ה-**אגנוסטיות הטכנולוגית**. 
בעוד שהתוספים המלווים מתאימים את הפלטפורמה *מבפנים*, האדפטרים ב-Core מתאימים אותה *מבחוץ*.
* **Plug & Play:** כל מערכת מיוצגת ב-Core על ידי Adapter. משתמשים יוכלו להחליף מודולים בקלות מבלי לשבור את הליבה.
* **תרגום שפה:** האדפטר מתרגם את ה-API של המערכת הספציפית למבנה נתונים אחיד וסטנדרטי שה-Core מבין.
* **חסינות לעתיד (Future-Proofing):** ארכיטקטורה זו מונעת "נעילת ספק" (Vendor Lock-in) ומבטיחה שהסטאק יתאים את עצמו לטכנולוגיות חדשות שיופיעו בעתיד.

## 📜 מפת הדרכים לפיתוח (Roadmap)

### שלב א': תשתית הליבה (הושלם ✅)
- הקמת מערך ה-Docker Compose הבסיסי.
- בניית ה-WordPress Adapter עם תמיכה באימות.
- מימוש יכולות CRUD דרך ה-Core.

### שלב ב': ה-Unified Stack וזהויות (בביצוע 🚧)
- פריסת מערכת IAM וחיבורה כ-Provider מרכזי.
- פיתוח התוסף המלווה (Companion Plugin) הראשוני לוורדפרס לחשיפת שדות API מתקדמים.
- פריסת קונטיינר NodeBB ו-MediaWiki כחלק מסביבת הפיתוח.

### שלב ג': אינטגרציה ופיתוח תוספים
- פיתוח אדפטרים ותוספים מלווים עבור MediaWiki ו-NodeBB.
- בניית לוגיקת אוטומציה: יצירת אובייקט ארכיוני מייצרת עקבות בכל פלטפורמות הקצה (פוסט, דף ויקי, ושרשור דיון).

### שלב ד': שחרור לקהילה (Open Source Release)
- בניית לוח בקרה (Dashboard) עצמאי.
- אריזת ה-Unified Stack להפצה קלה עם משתני סביבה גמישים.
- שחרור הקוד והתוספים תחת רישיון קוד פתוח.
